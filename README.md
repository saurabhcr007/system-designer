## вңЁ What is FlowForge?

FlowForge lets you **visually design system architectures** вҖ” drag and drop components like Load Balancers, Kubernetes clusters, Databases, and Redis caches onto a canvas, connect them, and then **run a live simulation** to watch packets flow through your system in real time.

Unlike static diagram tools, FlowForge:

- Routes packets **along the topology you draw** вҖ” not randomly
- Shows **per-node stress** with color-coded heat (green вҶ’ yellow вҶ’ orange вҶ’ red glow)
- Reacts dynamically when you **add or remove nodes mid-simulation**
- Works entirely in the browser вҖ” **no backend, no signup, no cost**

## рҹҺ¬ Features

### рҹ–ұпёҸ Visual Canvas
- Drag-and-drop components from the sidebar onto an infinite canvas
- Connect nodes by clicking source вҶ’ target to draw edges
- Group nodes inside **VPC**, **Kubernetes**, and **Subnet** container nodes
- Double-click any node to rename it inline
- Right-click context menu: Edit, Duplicate, Set Entry Point, Delete

### рҹ“Ұ Full Component Library

| Category | Components |
|---|---|
| **Frontend** | Web App, Mobile App, Tablet App, Desktop App, Smart Watch |
| **Traffic & Edge** | DNS, CDN, Load Balancer, API Gateway, WAF, Ingress, Reverse Proxy |
| **Compute** | Service, Container, Kubernetes Cluster, VM/Server, Lambda, Worker |
| **Storage** | PostgreSQL, MongoDB, Redis, Elasticsearch, S3/Blob, HDFS |
| **Networking** | VPC, Subnet, Firewall, VPN, NAT Gateway |
| **Messaging** | Kafka, RabbitMQ, SQS/Queue, Event Bus, WebSocket |
| **Monitoring** | Prometheus, Grafana, Logging, Alert Manager, APM/Tracing |
| **Security** | Auth Server/OAuth, Secret Manager, SSO/SAML |
| **AI & Agents** | AI Model, AI Agent, Vector DB, Embedding Service |

### рҹҺ¬ Live Simulation Engine
- **Topology-aware routing** вҖ” packets follow YOUR drawn edges via BFS path resolution
- **Fan-out support** вҖ” packets split at every branching node
- **Load Balancer awareness** вҖ” traffic distributes evenly across all connected downstream nodes
- **Dynamic rerouting** вҖ” add/remove nodes mid-sim and traffic adapts instantly
- **Configurable speed** вҖ” 0.5x to 3x simulation speed slider
- **Traffic load slider** вҖ” 1x to 10x traffic multiplier

### рҹҢЎпёҸ Node Heat Visualization
Every node has a live stress level (`currentRPS / maxCapacity`) displayed as:

| Stress | Color | Glow |
|---|---|---|
| 0вҖ“30% | рҹҹў Green `#22c55e` | Soft glow |
| 30вҖ“60% | рҹҹЎ Yellow `#eab308` | Moderate glow |
| 60вҖ“80% | рҹҹ  Orange `#f97316` | Strong glow |
| 80вҖ“100% | рҹ”ҙ Red `#ef4444` | Pulsing critical glow |

- Live **RPS badge** on each node during simulation
- **Overload warning** toast when a node hits 100% capacity
- 30% **packet drop** chance at overloaded nodes вҖ” increments error counter
- **Chaos Mode** вҖ” randomly fails nodes to simulate real-world outages

### рҹ’ҫ Export / Import / Share
- **Export JSON** вҖ” save your architecture as a portable `.json` file
- **Import JSON** вҖ” load any previously saved design
- **Share Link** вҖ” base64-encodes your design into the URL for instant sharing

## рҹҢҗ Deploy to GitHub Pages

### Step 1 вҖ” Configure Vite

In `vite.config.js`, set the base to your repo name:

```js
export default defineConfig({
  base: '/flowforge/',   // replace with your repo name
  plugins: [react()],
})
```

### Step 2 вҖ” Add deploy script to `package.json`

```json
{
  "homepage": "https://YOUR_USERNAME.github.io/flowforge",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
```

### Step 3 вҖ” Install gh-pages

```bash
npm install --save-dev gh-pages
```

### Step 4 вҖ” Deploy

```bash
npm run deploy
```

### Step 5 вҖ” Enable GitHub Pages

Go to your repo вҶ’ **Settings** вҶ’ **Pages** вҶ’ Source: `gh-pages` branch вҶ’ Save.

Your app will be live at `https://YOUR_USERNAME.github.io/flowforge` within 2 minutes.
