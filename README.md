## ‚ú® What is FlowForge?

FlowForge lets you **visually design system architectures** ‚ÄĒ drag and drop components like Load Balancers, Kubernetes clusters, Databases, and Redis caches onto a canvas, connect them, and then **run a live simulation** to watch packets flow through your system in real time.

Unlike static diagram tools, FlowForge:

- Routes packets **along the topology you draw** ‚ÄĒ not randomly
- Shows **per-node stress** with color-coded heat (green ‚Üí yellow ‚Üí orange ‚Üí red glow)
- Reacts dynamically when you **add or remove nodes mid-simulation**
- Works entirely in the browser ‚ÄĒ **no backend, no signup, no cost**

## ūüé¨ Features

### ūüĖĪÔłŹ Visual Canvas
- Drag-and-drop components from the sidebar onto an infinite canvas
- Connect nodes by clicking source ‚Üí target to draw edges
- Group nodes inside **VPC**, **Kubernetes**, and **Subnet** container nodes
- Double-click any node to rename it inline
- Right-click context menu: Edit, Duplicate, Set Entry Point, Delete

### ūüé¨ Live Simulation Engine
- **Topology-aware routing** ‚ÄĒ packets follow YOUR drawn edges via BFS path resolution
- **Fan-out support** ‚ÄĒ packets split at every branching node
- **Load Balancer awareness** ‚ÄĒ traffic distributes evenly across all connected downstream nodes
- **Dynamic rerouting** ‚ÄĒ add/remove nodes mid-sim and traffic adapts instantly
- **Configurable speed** ‚ÄĒ 0.5x to 3x simulation speed slider
- **Traffic load slider** ‚ÄĒ 1x to 10x traffic multiplier

### ūüĆ°ÔłŹ Node Heat Visualization
Every node has a live stress level (`currentRPS / maxCapacity`) displayed as:

| Stress | Color | Glow |
|---|---|---|
| 0‚Äď30% | ūüüĘ Green `#22c55e` | Soft glow |
| 30‚Äď60% | ūüü° Yellow `#eab308` | Moderate glow |
| 60‚Äď80% | ūüü† Orange `#f97316` | Strong glow |
| 80‚Äď100% | ūüĒī Red `#ef4444` | Pulsing critical glow |

- Live **RPS badge** on each node during simulation
- **Overload warning** toast when a node hits 100% capacity
- 30% **packet drop** chance at overloaded nodes ‚ÄĒ increments error counter
- **Chaos Mode** ‚ÄĒ randomly fails nodes to simulate real-world outages

### ūüíĺ Export / Import / Share
- **Export JSON** ‚ÄĒ save your architecture as a portable `.json` file
- **Import JSON** ‚ÄĒ load any previously saved design
- **Share Link** ‚ÄĒ base64-encodes your design into the URL for instant sharing

## ūüĆź Deploy to GitHub Pages

### Step 1 ‚ÄĒ Configure Vite

In `vite.config.js`, set the base to your repo name:

```js
export default defineConfig({
  base: '/FlowForge-Interactive-System-Designer/',   // replace with your repo name
  plugins: [react()],
})
```

### Step 2 ‚ÄĒ Add deploy script to `package.json`

```json
{
  "homepage": "https://saurabhcr007.github.io/FlowForge-Interactive-System-Designer",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
```

### Step 3 ‚ÄĒ Install gh-pages

```bash
npm install --save-dev gh-pages
```

### Step 4 ‚ÄĒ Deploy

```bash
npm run deploy
```

### Step 5 ‚ÄĒ Enable GitHub Pages

Go to your repo ‚Üí **Settings** ‚Üí **Pages** ‚Üí Source: `gh-pages` branch ‚Üí Save.

Your app will be live at `https://saurabhcr007.github.io/FlowForge-Interactive-System-Designer` within 2 minutes.
